## O que foi feito

Implementação completa da camada de rastreabilidade do projeto bci-calib.
Três módulos entregues, testados e aprovados pelo pipeline de pre-commit.

### logger.py
`configure_logging()` com structlog 24+. Processor chain:
`merge_contextvars → add_log_level → TimeStamper(utc) → StackInfoRenderer → ExceptionRenderer → ConsoleRenderer | JSONRenderer`.
Silencia `mne` e `moabb` para `WARNING` (nível 30) via
`logging.getLogger(lib).setLevel(logging.WARNING)`.
Seleção de renderer via `LOG_FORMAT=json` ou argumento explícito.
`structlog.reset_defaults()` antes de `configure()` garante idempotência.

### audit_db.py
SQLite com `PRAGMA journal_mode = WAL` e `PRAGMA foreign_keys = ON`.
4 tabelas STRICT: `experiment`, `run`, `metric`, `artifact`.
5 views: `v_run_experiment`, `v_latest_runs`, `v_run_metrics`,
`v_failed_runs`, `v_experiment_stats`.
Context manager `connection()` com commit/rollback automático.
Suporte a `:memory:` via conexão compartilhada (`self._memory_conn`).

### mlflow_tracker.py
`generate_correlation_id()` no formato `exp_{gitHash}_{ts}_{uid4}`.
`shutil.which("git")` resolve o executável antes de `subprocess.run`
(mitigação B607 — PATH hijacking). Fallback `"nogit"` sem exceção.
`MLflowTracker.start_run()` faz `bind_contextvars(correlation_id=cid)`
e seta tag `bci.correlation_id` no MLflow simultaneamente.

## Bugs encontrados e como foram resolvidos

**1. Arquivos em `src/bci_calib/infrastructure/` em vez de `src/infrastructure/`.**
Causa: instruções ambíguas sobre destino dos arquivos.
Resolução: mantida a estrutura Clean Architecture (`src/bci_calib/` como raiz);
todos os imports atualizados de `from infrastructure.*` para
`from bci_calib.infrastructure.*`.

**2. `table_count()` retornava 5 em vez de 4.**
Causa: `AUTOINCREMENT` cria `sqlite_sequence` interna visível em `sqlite_master`.
Resolução: `AND name NOT LIKE 'sqlite_%'` na query de contagem.

**3. `AuditDB(':memory:')` apagava tabelas entre chamadas.**
Causa: cada `sqlite3.connect(':memory:')` cria banco independente.
Resolução: `self._memory_conn` compartilhada quando `_is_memory = True`;
`connection()` retorna a mesma conexão sem fechar.

**4. `journal_mode()` retornava `'memory'` nos testes.**
Causa: WAL não é suportado em bancos in-memory pelo SQLite.
Resolução: fixture `file_db(tmp_path)` para o teste de WAL;
`:memory:` usa modo `memory` por design do SQLite.

**5. Typo `ppackages` no `pyproject.toml`.**
Causa: erro de digitação ao aplicar patch manualmente.
Resolução: corrigido para `packages = ["src/bci_calib", "src/infrastructure"]`
(depois simplificado — `infrastructure` ficou dentro de `bci_calib`).

**6. `test_v_latest_runs_ordering` não-determinístico.**
Causa: `DEFAULT (strftime('now'))` com precisão de 1 s; 3 inserts rápidos
colidiram no mesmo segundo tornando o `ORDER BY` não-determinístico.
Resolução: timestamps explícitos injetados via `INSERT ... VALUES (?, ts)`.

**7. Ruff E501 em 4 locais.**
Causa: linhas > 88 chars em docstring e queries SQL concatenadas.
Resolução: quebra vertical de argumentos e concatenação de string SQL.

**8. mypy `[misc]` nos decorators `@pytest.fixture`.**
Causa: `@pytest.fixture` não expõe generics completos nos stubs;
`disallow_untyped_decorators = true` (do `strict = true`) dispara falso positivo.
Resolução: `# type: ignore[misc]` nas duas fixtures afetadas.
Override `module = "tests.*"` no `pyproject.toml` não funcionou no Windows
(possível incompatibilidade de path separator com o módulo de matching do mypy).

**9. Bandit B603/B607 em `_git_short_hash`.**
Causa: `subprocess.run(["git", ...])` com path parcial vulnerável a PATH hijacking.
Resolução: `shutil.which("git")` resolve o executável completo;
`# nosec B603` com comentário de justificativa de segurança acima.

## Decisões de arquitetura

**Clean Architecture mantida.**
`infrastructure/` permanece sub-pacote de `bci_calib`, não pacote raiz separado.
Imports `from bci_calib.infrastructure.tracking.*` são explícitos e rastreáveis.

**Conexão compartilhada para `:memory:`.**
Decisão deliberada de tratar `:memory:` como caso especial no `__init__`.
Custo: um atributo extra (`_memory_conn`). Benefício: testes rápidos e isolados
sem precisar de `tmp_path` para cada teste de CRUD.

**`shutil.which` antes de subprocess.**
Elimina B607 sem remover a funcionalidade de incluir o git hash no CID.
Alternativa enterprise (injeção em build-time via `os.getenv("GIT_HASH")`)
adiada para quando o projeto tiver pipeline CI/CD hermético.

**Porta 5001 para MLflow UI.**
Porta 5000 ocupada no ambiente de desenvolvimento Windows.
Mitigação permanente via `just mlflow-ui` com `--port 5001`.
